⚠️ 部分対応
対応箇所状態方法

イベントページ（/events）✅Accept-Language自動検出。lib/i18n.ts経由
マップ・その他画面⚠️lib/i18n.ts + LangContextで部分対応
スポットデータ（name_en）❌新スキーマ移行後に対応
言語選択オンボーディング（S-00b）❌未実装

サーバーコンポーネントでの言語検出パターン

import { headers } from "next/headers";
import { detectLang, t } from "@/lib/i18n";

const headersList = await headers();
const lang = detectLang(headersList.get("accept-language") ?? "");
const tr = t[lang];

TypeScript 注意点

// ✅ 正しい（tはオブジェクトなのでReturnType不可）
tr: (typeof t)[keyof typeof t]

// ❌ 誤り（tはfunctionではない）
tr: ReturnType<typeof t[keyof typeof t]>

実装メモ・技術的決定事項

ジオコーディング: 国土地理院 API

ブラウザから直接叩くと User-Agent 制限に引っかかるため、 /api/geocode 経由でプロキシ。無料・APIキー不要。 日本国内のみ対応。Google Places APIへの移行で精度向上予定。

オーナーシップ判定: author_id（UUID）

author_email 比較ではなく author_id（UUID）を使用。 メールアドレスは変更・null になりうるが、IDは不変。

イベントページのキャッシュ（5分）

Google Sheetsのデータ変更後、最大5分間は古いデータが表示される。 { next: { revalidate: 300 } } で制御。 Vercel再デプロイでキャッシュは即時クリアされる。
